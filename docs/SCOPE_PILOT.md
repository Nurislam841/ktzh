# Scope пилота (Точка А)

## Границы пилота
- Узел: 1 станция (по умолчанию `Алматы-1`) + направление до следующей станции.
- Пути: 8 станционных путей.
- Окно планирования: 6 часов (переключаемое 6/12/24 на экране узла).
- Типы поездов: пассажирские и грузовые (приоритеты 1/2/3 через enum PASSENGER/FREIGHT/OTHER).

## Сценарии пилота
1. Формирование поезда: готовый состав -> запрос ресурсов -> назначение -> отправление.
2. Транзит/смена: прибытие -> смена бригады/локомотива (при необходимости) -> новый слот -> отправление.

## Ресурсы в расчете
- Локомотив
- Бригада
- Путь

## События для пересчета
- LOCO_FAILURE
- CREW_UNAVAILABLE
- TRAIN_DELAY
- TRACK_BLOCKED
- MAINTENANCE_STARTED / MAINTENANCE_ENDED
- WEATHER
- CAPACITY_CONFLICT

## Техническая цель пилота
При событии система создает новую версию расписания, фиксирует изменения и показывает diff "до/после".

## Определение готовности пилота
- Можно загрузить данные (mock или импорт).
- Можно вызвать событие и получить новую версию графика.
- Есть проверка конфликтов по бригаде/локомотиву/пути.
- Есть дашборд и экран сравнения версий.
